# Contributing to Awesome Clawdbot

Thank you for your interest in contributing! This document provides guidelines for contributing to this awesome list.

## How to Contribute

### Adding a New Use Case

1. Fork this repository
2. Create a new branch: `git checkout -b add-use-case-name`
3. Add your use case following the format below
4. Submit a Pull Request

### Format for Use Cases

```markdown
#### Use Case Title

Brief description of what this achieves (1-2 sentences).

**Prompt Template:**

\`\`\`markdown
Your prompt here with [PLACEHOLDERS] for customization.
Include all necessary context the user should provide.
\`\`\`

**What You Get:**
- Expected output 1
- Expected output 2
- Expected output 3

**Tips:** (optional)
- Any helpful tips for better results
```

### Quality Guidelines

- **Test your prompts**: Make sure they work with Clawdbot before submitting
- **Be specific**: Vague prompts lead to vague results
- **Sanitize data**: Never include personal or sensitive information
- **Use English**: All contributions should be in English
- **One PR per item**: Unless items are closely related
